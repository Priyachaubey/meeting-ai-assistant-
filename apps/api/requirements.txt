# passlib 1.7.4 reads bcrypt's removed `__about__.__version__` attribute to detect the
# installed bcrypt version. That attribute was removed in bcrypt 4.1.0 (Oct 2023) — passlib
# itself hasn't shipped a fix (1.7.5 has been unreleased for years). Without this pin, `pip
# install` resolves to whatever the latest bcrypt is (5.x as of writing), and every single
# password hash/verify call — i.e. every login and registration — throws
# "AttributeError: module 'bcrypt' has no attribute '__about__'" the moment CryptContext
# touches it. `bcrypt[bcrypt]==1.7.4` alone does NOT pin the bcrypt package version, only
# declares it as a dependency — this explicit pin is what actually fixes it.
